# ADR 0001: ADR 도입

- 날짜: 2026-05-15
- 상태: 채택됨 (Accepted)
- 결정자: 병승

## 컨텍스트

팀원이 주니어라 의사결정의 배경이 흐릿하게 사라지면 같은 논의를 반복하게 된다. 부재 시 일관된 기준으로 판단하려면 결정 기록이 필요하다.

## 결정

`docs/decisions/`에 ADR을 두고 모든 기술 결정/인터페이스 변경/룰 변경을 기록한다. 번호는 4자리 (`0001`, `0002`, ...). 템플릿은 `TEMPLATE.md`.

## 결과

- 좋은 점:
  - "왜 이렇게 짰지?"에 6개월 뒤 답할 수 있다.
  - 새 팀원/fincoach-bot이 컨텍스트 빠르게 잡는다.
  - PR 리뷰 시 "이 변경 ADR 있나?" 한 줄로 검증 가능.
- 트레이드오프: 작은 결정에도 문서 작성 부담. → "shared/ 변경 또는 라이브러리 교체급" 이상에만 ADR 강제.

## 대안

- 회의록만으로 충분 → 검색 어렵고 의사결정만 골라내기 힘듦. 기각.
- 노션에만 기록 → 코드 컨텍스트랑 분리됨. fincoach-bot이 접근 어려움. 기각.
