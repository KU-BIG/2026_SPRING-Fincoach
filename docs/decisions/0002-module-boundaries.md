# ADR 0002: 모듈 경계

- 날짜: 2026-05-15
- 상태: 채택됨 (Accepted)
- 결정자: 병승, 은서, 현태, 수빈 (4월 노션 회의에서 합의)

## 컨텍스트

3명의 팀원이 6주 안에 통합 가능한 결과물을 만들어야 함. 모듈 경계가 모호하면 인터페이스 충돌 + 머지 지옥. 반대로 너무 잘게 쪼개면 단일 책임이 흐트러지고 통합 부담 큼.

## 결정

3개 핵심 모듈 + 1개 공통 모듈로 분리:

- `market_intelligence/` (은서): 외부 시장 데이터 수집/가공 → `MarketOutput`
- `portfolio_analyzer/` (현태): 사용자 포트폴리오 분석/백테스트 → 4개 dict 반환 함수
- `coach_chat/` (수빈): Streamlit UI + LLM 챗
- `shared/` (병승): 데이터 계약, mock, config

각 팀원은 자기 폴더만 수정. `shared/`는 공통 영역(CODEOWNERS 자동 라우팅).

## 결과

- 좋은 점:
  - 일찍 인터페이스만 합의하면 각자 독립 개발 가능.
  - mock 덕에 다른 모듈 안 기다리고 진행.
  - 머지 충돌 거의 없음 (서로 다른 폴더).
- 트레이드오프: 모듈 간 의존성 변경이 필요할 때마다 인터페이스 협의 필요. → ADR로 대응.

## 대안

- 단일 모놀리식: 충돌 너무 잦을 것. 기각.
- 마이크로서비스: 6주 안에 인프라 부담. 기각.
