# PR 리뷰 플레이북 (CTO/cto-bot 공용)

## 리뷰는 30분 이내

큰 리뷰는 PR을 쪼개라고 요청. 30분 넘게 걸리면 "이거 너무 큼. 두세 개로 쪼개주세요" 코멘트.

## 우선순위 (순서대로)

### 1. 안전/법적 (블로커 - 무조건 NO-GO)
- 매수/매도 직접 추천 표현 (`매수`, `매도`, `buy this`, `sell this`)
- 면책 문구 누락
- 위험 git 명령 (force push, reset --hard)
- 시크릿 노출 (.env, API 키)

### 2. 룰 위반 (대부분 NO-GO)
- main 직접 commit (hook으로 막혔어야 함)
- 다른 사람 모듈 폴더 수정
- `shared/` 변경에 ADR 없음
- 커밋 prefix 누락

### 3. 계약 위반 (NO-GO)
- `shared.models` 우회하고 dict 직접 사용
- type hint 누락
- mock과 실 데이터 둘 중 하나만 작동

### 4. 품질 (REQUEST CHANGES, 협의)
- 테스트 없음
- 외부 API 호출에 에러 처리 없음
- 캐시 없이 매번 호출
- 디버그 코드 남음 (`print`, `breakpoint`)
- 데드 코드, 사용 안 하는 import

### 5. 스타일 (코멘트로 제안)
- 변수명 모호
- 함수 너무 김 (50줄+)
- 주석 과다 (코드가 자명하면 주석 빼라)

## 코멘트 톤

- 명령형보다 질문형: "이거 빼는 게 어때요?" 보다 ❌ → "이게 왜 필요한지 알려줄 수 있어요?" ✓
- 이유 항상 첨부: "X 수정" 보다 ❌ → "X 수정. Y 때문에 Z 발생함" ✓
- 잘된 점도 코멘트: "이 mock 깔끔하네요" 같이 한 줄이라도

## APPROVE 조건

다음 모두 충족:
- [ ] CI 통과
- [ ] 안전/법적 항목 클린
- [ ] 룰 위반 없음
- [ ] 계약 준수
- [ ] 최소 1개 테스트 (happy path)
- [ ] 본인 모듈만 수정 (shared/는 ADR 첨부 시 OK)

## REQUEST CHANGES 조건

위 항목 중 하나라도 빠짐. 코멘트에 구체 항목 나열.

## COMMENT 조건

작동은 하지만 개선 여지 있음. 머지 후 follow-up 가능한 사안.

## 표준 코멘트 템플릿

### PR 너무 큼
> 변경 범위가 커서 안전하게 리뷰가 어렵습니다. 다음 두 개로 쪼개주세요:
> 1. {파트 A}
> 2. {파트 B}

### shared/ 변경에 ADR 없음
> `shared/models.py` 변경이 있는데 ADR이 없네요. `docs/decisions/TEMPLATE.md` 복사해서 ADR 먼저 올려주세요. 영향 모듈: {은서, 현태, 수빈}.

### 매수/매도 표현
> "{발견한 문장}" 부분이 매수/매도 직접 추천처럼 보입니다. "정보 제공" 톤으로 바꿔주세요. 예: "이 종목은 X 특성이 있습니다" (O), "이 종목 매수 권장" (X). 법적 리스크 때문입니다.

### 면책 누락
> LLM 출력 반환 전에 `shared.disclaimers.attach()` 호출 추가 필요합니다.

### mock과 실 데이터 둘 다 통과 확인
> 이 함수 mock으로만 테스트된 것 같은데, 실 데이터에서도 동작하는지 한 번 확인 부탁드립니다. 외부 API 응답 형식이 mock과 다를 수 있어요.

## cto-bot 호출

본인이 시간 없거나 큰 PR이면 cto-bot에게 1차 리뷰 위임:

```
claude /cto-review {PR번호}
```

cto-bot이 위 체크리스트 돌리고 코멘트 남김. 본인은 그 위에 최종 판단만.
