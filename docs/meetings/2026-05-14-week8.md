## Claude Context 문제 해결책

**문제**: 매일 새 채팅 시작하면 어제 작업한 컨텍스트가 다 사라짐.

**해결**:
- md 파일로 매일 개발한 내용 + 채팅 내용 요약
- md 파일 기반으로 다시 새 채팅 시작
- 매일 개발하고 → 기록 남기고 → 깃 올리기 **자동화 skill 필요**

→ FinCoach 0주차에 `wrap-day`, `start-day` 스킬로 구현됨 (`docs/daily-logs/{본인}/`).

### MVP 1단계 (6 작업)
1. JSON 스키마 확정
2. 종목명 → 티커 자동 매칭
3. 현재가/변동률 조회 (한국=pykrx / 미국=yfinance)
4. 수익률/평가금액/비중 계산 (핵심 차별점)
5. Streamlit 요약 표
6. 출력 함수 `get_portfolio_data() -> dict` 확정

### 최종 2단계 (8 작업)
7. 섹터 자동 분류
8. 원형 차트 4종
9. 종목 상세 - 주가 차트 (매수평단가 수평선)
10. 종목 상세 - 뉴스/키워드 (은서 모듈 연결)
11. 백테스트
12. 벤치마크 + 성과 지표 (CAGR/MDD/Sharpe, QQQ/KOSPI)
13. LLM 분석 리포트
14. 은서/수빈 실연결 + 통합 테스트

### 수빈과 연결 인터페이스 (현태 결정)

| 항목 | 선택 | 이유 |
|------|------|------|
| 데이터 포맷 | dict 반환 | LLM context로 바로 `json.dumps()` |
| 함수 구조 | 분리 (4개) | 챗봇이 필요한 것만 호출 → LLM 비용 절약 |
| 갱신 타이밍 | 세션 캐시 | 발표용 충분, 빠름 |
| analysis_report | 구조화된 dict | "리스크만 알려줘" 같은 질문에 정확 응답 |
| 차트 | plotly figure 객체 | 수빈이 `st.plotly_chart(fig)` 한 줄로 끝 |

```python
def get_portfolio_data() -> dict: ...           # 빠름, 항상 OK
def get_analysis_report(force_refresh=False) -> dict: ...  # 느림, 요청 시
def get_backtest_result(period="1y") -> dict: ...          # 느림, 요청 시
def get_stock_chart(ticker, period="1mo") -> plotly.Figure: ...
```
